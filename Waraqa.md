# اسم المشروع Waraqa 
---

## نظرة عامة على المشروع

محرر LaTeX تعاوني عبر الإنترنت مشابه لـ Overleaf، يتيح كتابة وتجميع وعرض مستندات LaTeX مباشرةً من المتصفح دون الحاجة إلى تثبيت أي برنامج.

---

## المتطلبات الوظيفية

### 1. المحرر (Editor)
- تلوين صياغة LaTeX (Syntax Highlighting)
- إكمال تلقائي للأوامر والبيئات (`\begin{}`, `\frac{}{}`, ...)
- إظهار أرقام الأسطر
- إظهار الأخطاء النحوية فور الكتابة (Linting)
- دعم اختصارات لوحة المفاتيح (Vim / Emacs اختياري)
- طي الكتل (Code Folding)

### 2. التجميع (Compilation)
- تجميع تلقائي عند الحفظ أو بعد توقف الكتابة
- دعم محركات متعددة: `pdfLaTeX`, `XeLaTeX`, `LuaLaTeX`
- عرض سجل أخطاء التجميع مع رقم السطر
- إعادة تجميع فقط الملفات التي تغيرت (Incremental Compilation)
- حد زمني قابل للتعديل لكل تجميع

### 3. عارض PDF (PDF Viewer)
- عرض مباشر بجانب المحرر (Split View)
- مزامنة ثنائية الاتجاه (SyncTeX): النقر على النص ← يقفز إلى السطر في المحرر، والعكس
- تكبير / تصغير وتدوير الصفحات
- بحث داخل PDF

### 4. إدارة الملفات (File Manager)
- هيكل شجري للمشاريع والملفات
- دعم ملفات متعددة (`.tex`, `.bib`, `.sty`, `.cls`, صور...)
- رفع ملفات وصور
- نسخ احتياطي تلقائي

### 5. التعاون (Collaboration)
- تحرير مشترك في الوقت الفعلي (Real-time Collaboration)
- تتبع التغييرات مع اسم المستخدم
- تعليقات على النص
- إدارة صلاحيات (مالك / محرر / قارئ)
- مشاركة رابط مباشر

### 6. إدارة المستخدمين
- تسجيل وتسجيل دخول (OAuth + بريد إلكتروني)
- لوحة مشاريع شخصية
- تاريخ المراجعات (Git-based)
- إعدادات المحرر محفوظة للمستخدم

## مكونات النظام التفصيلية

### الواجهة الأمامية (Frontend)

| المكون | التقنية المقترحة |
|---|---|
| الإطار العام | React أو Vue 3 |
| محرر الكود | Monaco Editor أو CodeMirror 6 |
| عرض PDF | PDF.js |
| التزامن الفوري | Y.js + WebSocket (CRDT) |
| إدارة الحالة | Zustand أو Pinia |
| التصميم | Tailwind CSS |

### الواجهة الخلفية (Backend)

| المكون | التقنية المقترحة |
|---|---|
| الخادم الرئيسي | Node.js + Express أو Fastify |
| خادم التجميع | Python أو Node.js Worker |
| التزامن الفوري | Socket.IO أو WebSocket native |
| قاعدة البيانات | PostgreSQL + MongoDB |
| التخزين المؤقت | Redis |
| تخزين الملفات | MinIO (self-hosted) أو AWS S3 |
| المصادقة | JWT + OAuth 2.0 |

### خادم التجميع (Compile Server)

```
طلب التجميع
      │
      ▼
قائمة انتظار (Redis Queue)
      │
      ▼
Worker يسحب المهمة
      │
      ▼
Docker Container معزول
  ├── نسخ ملفات المشروع
  ├── تشغيل pdflatex / xelatex
  ├── جمع السجلات والأخطاء
  └── رفع PDF إلى S3
      │
      ▼
إرسال النتيجة للعميل عبر WebSocket
```
